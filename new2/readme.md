新增功能

  1. 分钟价格信号函数 (get_minute_price_increase_signal)：
    - 比较当前市场价格与1分钟前的收盘价
    - 如果价格上涨超过0.1%，产生 BUY 信号
    - 使用 client.get_symbol_ticker() 获取实时价格，get_historical_klines() 获取1分钟前价格
  2. 主循环更新 (main 函数)：
    - 每分钟检查一次分钟价格信号
    - 当信号为 BUY 时，固定买入 0.001 BTC
    - 自动检查USDT余额是否足够
    - 保留原有MA20策略，每5分钟执行一次
  3. 订单数量限制调整 (place_order 函数)：
    - 将最小交易量从0.01 BTC改为0.00001 BTC（支持0.001 BTC交易）
    - 数量四舍五入到5位小数以匹配Binance精度

  策略逻辑

  - 触发条件：当前价格 > 1分钟前价格 × 1.001
  - 买入数量：固定 0.001 BTC
  - 执行频率：每分钟检查一次
  - 风险控制：检查USDT余额，不足时跳过

  注意事项

  1. API限制：每分钟调用2次API（价格查询 + K线数据），注意测试网频率限制
  2. 重复触发：价格持续上涨可能导致多次买入，可考虑添加冷却机制
  3. 最小交易量：Binance现货BTCUSDT最小交易量为0.00001 BTC，0.001 BTC符合要求
  4. 余额检查：买入前会验证USDT余额，避免资金不足
  5. 测试环境：代码使用 testnet=True，确保API密钥有测试网络权限

  文件结构

  trade_en.py
  ├── get_minute_price_increase_signal()  # 新增分钟策略
  ├── get_trade_signal()                  # 原有MA20策略
  ├── place_order()                       # 修改最小交易量
  └── main()                              # 双策略并行执行

  现在程序将同时运行两种策略：
  - 每分钟：检查价格是否上涨0.1%，是则买入0.001 BTC
  - 每5分钟：检查MA20交叉信号，执行原有风险控制交易
